Chapter 1

What is Programming?

Programming is the art of giving very detailed, step-by-step instructions to a computer — like giving precise directions to someone who has never visited your house.

Key IdeaComputers are incredibly literal. They can't fill in gaps or make assumptions — every instruction must be exact.
Chapter 1

Programming is Like a Recipe

Traditional Recipe"Add some flour and mix until it looks right"
Programming Recipe1. Add exactly 2 cups of flour
2. Mix for 3 minutes at medium speed
3. If too dry, add 1 tbsp of water
4. Repeat step 3 until consistent
Chapter 2

Understanding Programming Languages

A programming language is a formal system for giving instructions to a computer. Like human languages, each has grammar, vocabulary, and syntax — designed for a specific purpose.

  • Hammer (C++) — powerful and precise, requires skill
  • Electric Drill (Python) — easier, versatile, great for many tasks
  • Screwdriver (JavaScript) — specialized but excellent at its domain

Chapter 3

By Programming Paradigm

  • Procedural — step-by-step recipe, focus on sequence. (C, Pascal)
  • Object-Oriented (OOP) — organise code into objects with properties & behaviors. (Java, C++, Python)
  • Functional — transform data through mathematical functions. (Haskell, Lisp)

Python, JavaScript, Ruby
Chapter 4

Introduction to Python

Why Python?Python is like the Swiss Army knife of programming — versatile, reliable, and easy to use. Created by Guido van Rossum in 1991 with one goal: code should be readable.
  • Readability — code looks almost like English
  • Versatility — web, data science, AI, automation, games
  • Beginner-Friendly — handles complexity so you can focus on solving problems
  • Huge Community — millions of developers, endless resources

Chapter 5

How Python Executes Code

Compiled (Meal Kit)Source → Compiler → Machine Code → Execution.
Pros: very fast. Cons: recompile for every change.
Interpreted (Personal Chef)Source → Interpreter → Real-time execution.
Pros: immediate feedback, platform-independent.
Chapter 5

Python's Hybrid Approach

Best of Both WorldsPython first compiles your code to bytecode (.pyc), then the Python Virtual Machine interprets it — giving reasonable speed with the flexibility of interpretation.
  • .py files — your human-readable source code
  • .pyc files — intermediate bytecode (like mise en place)
  • Python VM — reads and executes the bytecode
Chapter 6

Installing Python on Windows

  • Step 1 — Visit python.org → Downloads → get the latest Python 3.x.x

Example: Password Validator

password_validator.py
def validate(password):
    has_upper = has_lower = has_digit = False
    for char in password:
        if   char.isupper(): has_upper = True
        elif char.islower(): has_lower = True
        elif char.isdigit(): has_digit = True
    return has_upper and has_lower and has_digit and len(password) >= 8

for pwd in ["Password123", "password", "Pass123"]:
    s = "strong ✓" if validate(pwd) else "weak ✗"
    print(f"'{pwd}' → {s}")

Example: Guessing Game

guessing_game.py
import random

target = random.randint(1, 10)
guess = 0 ; attempts = 0

while guess != target and attempts < 3:
    guess     = random.randint(1, 10)
    attempts += 1
    if   guess == target: print(f"Correct in {attempts}!")
    elif guess <  target: print(f"{guess} is too low")
    else:                  print(f"{guess} is too high")

if guess != target:
    print(f"The number was {target}")
